EDA (Exploratory data analysis)

What is EDA?â€‹

EDA is the process of performing initial investigations on data so as to discover patterns, to spot anomalies, to test hypotheses with the help of summary statistics and graphical representations. In EDA there different stages for data cleaning and imputation of missing values, Handling outliers and calculating different statistics. Date feature will be manipulated as per date change format.

Steps to perform eda for predictive modelingâ€‹

  1. Goto EDA
  2. Select dataset and target feature.
  3. Click on Start EDA button. It shows the all the process one by one in EDA stages popup window
  4. Click on continue button.

Steps to perform eda for clusteringâ€‹

  1. Goto EDA
  2. Select dataset
  3. Click on Start EDA button. It shows the all the process one by one in EDA stages popup window
  4. Click on continue button

Steps to perform eda for timeseriesâ€‹

  1. Goto EDA
  2. Select dataset, index and target feature
  3. Click on Start EDA button. It shows the all the process one by one in EDA stages popup window
  4. Click on continue button

Steps to perform eda for multiple timeseriesâ€‹

  1. Goto EDA
  2. Select dataset, index ,target feature and timeseries id
  3. Click on Start EDA button. It shows the all the process one by one in EDA stages popup window
  4. Click on continue button

Feature analysisâ€‹

It shows graphical representation of features with statistical details. With more than one feature the comparison of different features in the form of bar plot, box plot, scatter plot, pie plot, histogram plot, word cloud, 2D and 3D plots etc.

Steps to check the feature analysis plotâ€‹

  1. Goto EDA
  2. Click on feature analysis button
  3. Select features from dropdown
  4. Click on plot option radiobutton
  5. Click on submit button it will give detailed visualization and comparison of data

Share reportâ€‹

In EDA Module share report option is enable after performing eda operation.

Steps to Share the reportâ€‹

  1. Goto EDA
  2. Select dataset and target feature
  3. Click on Start EDA button
  4. Click on continue button to close the EDA stages popup window
  5. Click on share report button. it will redirect to reports module
  6. Click on share with user button and pass the email id. Using share with user feature you can share the report with different user like developer, report user, admin user role
  7. Click on share button

Based on target feature selection in eda module you can see the different plots in target distribution.

EDA advanced operationsâ€‹

Split data setâ€‹

The data should ideally be divided into 3 sets â€“ namely, train, test, and holdout cross-validation or development (dev) set. Letâ€™s first understand in brief what these sets mean and what type of data they should have.

Train setâ€‹

The sample of data used to fit the model that is the actual subset of the dataset that we use to train the model (estimating the weights and biases in the case of Neural Network). The model observes and learns from this data and optimize its parameters.

Dev setâ€‹

The development set is used to validate the trained model. This is the most important setting as it will form the basis of our model evaluation. If the difference between error on the training set and error on the dev set is huge, it means the model as high variance and hence, a case of over-fitting.

Test setâ€‹

The test set contains the data on which we test the trained and validated model. It tells us how efficient our overall model is and how likely is it going to predict something which does not make sense. There are a plethora of evaluation metrics (like precision, recall, accuracy etc.) which can be used to measure the performance of our model.

Date wiseâ€‹

Only for predictive modelling the data containing date feature user can split the dataset. This split happens and data get divided in above the sets. It is an optional feature.

Missing thresoldâ€‹

Missing thresold sets the thresold value for number of missing values. If the number of missing values exceeds the thresold values, then the respective column would be dropped from the dataset.

Imputation featureâ€‹

If user knows target variable in advance then selecting that as imputation feature, all the operations of manual eda are handled according to the imputation feature. For example - suppose you have age and gender in your data. and there are some missing values in age. We want to fill the values in age using gender. i.e. if it is a male, we want to compute average of all male ages and fill that missing value.

Conditional filteringâ€‹

It is the process to filter/remove the unwanted data from the dataset. Conditional filtering is essentially filtering out your data so that youâ€™re left with data that meets a certain criteria. It can be extremely convenient for viewing specific values and stripping back anything not useful or removing values that simply do not answer your question. Example cases could be if you only want to view Premier League teams that had scored more than 40 goals that season, or youâ€™re only interested in seeing movies that have grossed over 500 million USD. Conditional filtering allows you to do exactly that.

String transformationâ€‹

String transformation is the method to change a string value in a column to another desired value. It will reflect in training and pipeline preview in models.

Resamplingâ€‹

Resampling involves changing the frequency of your time series observations. It is basically means representing the data with a different frequency. In time series, data consistency is of prime importance, resampling ensures that the data is distributed with a consistent frequency. Resampling can also provide a different perception of looking at the data, in other words, it can add additional insights about the data based on the resampling frequency.
